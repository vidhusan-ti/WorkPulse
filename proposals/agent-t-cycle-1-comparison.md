## Key Findings

- **#2 CTA** is the most accurate single signal — best standalone approach
- **#4 EJAD** has zero false positives but misses soft genuine insight (false negative risk)
- **#6 SND** is excellent as a pre-filter but promotes novelty without validating impact
- **#8 IOAS** is sharp on intentional steering; misses exploratory/unclear intent prompts
- **#11 RAGC** is the best long-term investment but needs a calibrated corpus first

## Best Combination Pipeline

```
─────────────────────────────────────────────
WORKPULSE GRADING PIPELINE — COMBINATION v1
─────────────────────────────────────────────

Stage 1: HARD EXCLUSION FILTER
  Approach:  #6 SND (Semantic Novelty Detection)
  Purpose:   Eliminate obvious below-bar cheaply
  Cost:      Very low — embedding similarity, no LLM call
  Threshold: Novelty score < 0.25 → BELOW-BAR, stop
  Catches:   "yes", pure confirmations, zero-distance responses

Stage 2: INTENT + STEERING CHECK
  Approach:  #8 IOAS (Intent-Outcome Alignment)
  Purpose:   Confirm user was steering, not just responding
  Cost:      Medium — single LLM call
  Threshold: Intent clarity × outcome precision < 0.45 → BELOW-BAR
  Catches:   Operational prompts that passed Stage 1

Stage 3: TRAJECTORY CONFIRMATION
  Approach:  #2 CTA (Conversation Trajectory Analysis)
  Purpose:   Confirm conversation actually improved because of this prompt
  Cost:      Medium — requires next 2-3 turns
  Logic:     Strong trajectory delta → ABOVE-BAR candidate
             Weak delta → NEAR-BAR, hold for Stage 4
  Catches:   Clear intent but weak LLM follow-through

Stage 4: ADVERSARIAL VALIDATION (above-bar candidates only)
  Approach:  #4 EJAD (Ensemble + Adversarial Dissenter)
  Purpose:   Final false-positive check before portfolio approval
  Cost:      High — 3 LLM calls
  Logic:     Survives dissenter attack → ABOVE-BAR confirmed
             Dissenter wins → downgrade to NEAR-BAR
  Catches:   Edge cases that look above-bar but aren't

Stage 5: HISTORICAL CALIBRATION (Phase 2)
  Approach:  #11 RAGC (Retrieval-Augmented Grade Calibration)
  Purpose:   Anchor decisions to real historical examples
  Cost:      Medium — vector DB retrieval + LLM
  Activates: Once corpus reaches 50+ graded windows
  Impact:    Reduces drift, improves consistency over time

─────────────────────────────────────────────
FINAL DECISION LOGIC
─────────────────────────────────────────────
  Stage 1 fail → BELOW-BAR (silent)
  Stage 2 fail → BELOW-BAR (silent)
  Stage 3 weak → NEAR-BAR (coaching popup)
  Stage 4 fail → NEAR-BAR (coaching popup)
  Stage 4 pass → ABOVE-BAR (approval popup)
─────────────────────────────────────────────
```
